def skin_detect(frame):
    hsv = cv2.cvtColor(frame, cv2.COLOR_BGR2HSV)
    lower_skin = np.array([0, 20, 70], dtype=np.uint8)
    upper_skin = np.array([20, 255, 255], dtype=np.uint8)
    mask = cv2.inRange(hsv, lower_skin, upper_skin)

### 13:00 - 15:00：手指状态检测

检测问题解决了，现在要解决的是手势识别问题。

MediaPipe返回的是21个关键点的坐标。我需要根据这些坐标来判断每个手指的状态——伸直还是弯曲。

这听起来很简单，做起来也很简单，但做对的很难。

首先定义关键点：

- 拇指：0, 1, 2, 3, 4
- 食指：0, 5, 6, 7, 8
- 中指：0, 9, 10, 11, 12
- 无名指：0, 13, 14, 15, 16
- 小指：0, 17, 18, 19, 20

其中，带4的是指尖，带2/5/9/13/17的是各手指的基关节。

判断手指状态的逻辑：

```python
def get_finger_state(landmarks):
    states = []
    tips = [4, 8, 12, 16, 20]
    bases = [2, 5, 9, 13, 17]
    
    # 拇指比较特殊，需要横向判断
    thumb = landmarks[tips[0]]
    thumb_base = landmarks[bases[0]]
    states.append(1 if thumb.x > thumb_base.x else 0)
    
    # 其他手指纵向判断
    for i in range(1, 5):
        tip = landmarks[tips[i]]
        base = landmarks[bases[i]]
        states.append(1 if tip.y < base.y else 0)  # y越小越靠上
    
    return states
```

对于拇指，我用x坐标判断（因为拇指是横着的）；对于其他手指，用y坐标判断（因为其他手指是竖着的）。

这个方法有个问题：当手掌面对摄像头时，y坐标判断会有问题。

因为MediaPipe的坐标系统是归一化的，y越小表示越靠上。

所以如果指尖比基关节更靠上（y更小），说明手指伸直了。

但这个假设只有在手掌面对摄像头、背对背景时才成立。

如果手掌侧面对着摄像头，这个判断就会出问题。

不过，对于我现在的应用场景——正对摄像头做手势——这个方法足够了。

先凑合着用吧。

### 15:00 - 16:30：平滑算法

手势识别最怕的就是抖动。

想象一下，你只是想做个"手枪"手势（食指伸直，其他手指弯曲），结果因为手抖或者其���原因，程序把你短暂地识别成了"胜利"手势（食指和中指伸直）。

这对于机械臂控制来说是灾难性的。

因为机械臂会在这短暂的时间内执行错误的命令。

所以我需要一个平滑算法。

最简单的办法是使用缓冲区：

```python
finger_states_buffer = []

# 每帧获取到手指状态后
finger_states_buffer.append(tuple(finger_states))
if len(finger_states_buffer) > 5:
    finger_states_buffer.pop(0)

# 计算平均状态
avg_state = [0]*5
for fs in finger_states_buffer:
    for i in range(5):
        avg_state[i] += fs[i]
for i in range(5):
    avg_state[i] = 1 if avg_state[i] >= len(finger_states_buffer)/2 else 0
```

保留最近5帧的状态，然后取多数投票。

如果5帧中有3帧及以上是同一个状态，就使用那个状态。

这样做的好处是：即使偶尔有1-2帧检测错误，也不会影响最终结果。

同时，我还需要一个稳定时间的要求：

```python
if last_gesture == gesture:
    stable = min(stable + 1, 3)
else:
    stable = 0
last_gesture = gesture
```

同一个手势需要连续出现3次，才认为是稳定的。

只有稳定的手势才会发送控制命令。

这套组合拳打下来，抖动问题基本解决了。

虽然偶尔还是会有误判，但已经在我能接受的范围内了。

要求不能太高，在这个项目上，我已经学会了"差不多就行"。

### 16:30 - 18:00：串口通信

检测和识别都搞定了，现在要让机械臂动起来。

首先需要找到正确的串口。

在Windows上，串口名称通常是COM1、COM2、... COMn。

但具体是哪个口，取决于你的USB连接。

我的机械臂连接在COM5上——这是试出来的。

代码中我尝试了多个可能的串口：

```python
for port in ['COM5', 'COM6', 'COM7', 'COM8', 'COM9', 'COM3', 'COM4']:
    try:
        ser = serial.Serial(port, 115200, timeout=1)
        print(f"Connected to {port}")
        time.sleep(2)
        break
    except:
        continue
```

波特率是115200。这是和Arduino开发板通信常用的波特率。

发送命令的格式是：舵机编号 + 角度 + 回车换行。

例如：`1` `180` `\r\n` 表示"舵机1转到180度"。

```python
def move(s, a):
    if ser:
        try:
            ser.write(f"{s}{a}\r\n".encode())
        except:
            pass
```

简单粗暴，但有用。

现在每根手指对应一个舵机：手指伸直对应180度，手指弯曲对应0度。

这是个粗略的映射，但作为原型，足够了。
